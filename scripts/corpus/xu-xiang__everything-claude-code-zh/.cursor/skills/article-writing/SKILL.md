# 文章写作 (Article Writing)

撰写听起来像真人或品牌、而非平庸 AI 输出的长篇内容。

## 适用场景 (When to Activate)

- 起草博客文章、短评 (Essays)、发布公告、指南、教程或时事通讯 (Newsletter)
- 将笔记、转录文本或研究资料转化为精炼的文章
- 从示例中匹配现有的创始人、运营者或品牌语调
- 优化已撰写长篇文案的结构、节奏和论据

## 核心规则 (Core Rules)

1. 以具体事物开头：示例、输出、轶事、数字、截图描述或代码块。
2. 在示例之后进行解释，而非之前。
3. 优先使用简短、直接的句子，避免冗长。
4. 使用有来源支持的具体数字。
5. 严禁捏造传记事实、公司指标或客户证据。

## 语感提取工作流 (Voice Capture Workflow)

如果用户需要特定语调，请收集以下一项或多项素材：
- 已发表的文章
- 时事通讯 (Newsletters)
- X / LinkedIn 帖子
- 文档或备忘录
- 简短的风格指南 (Style Guide)

然后提取：
- 句长和节奏
- 语调是正式、对话式还是犀利的
- 偏好的修辞手段，如括号、列表、碎片句或提问
- 对幽默、观点和非主流框架 (Contrarian Framing) 的接受度
- 格式习惯，如标题、项目符号、代码块和引用句 (Pull Quotes)

如果没有提供语调参考，默认使用直接的“运营者风格”：具体、实用且低炒作。

## 禁用模式 (Banned Patterns)

删除并重写以下任何内容：
- 平庸的开场白，如“在当今快速演变的格局中...”
- 填充性过渡词，如“Moreover”和“Furthermore”
- 炒作性词汇，如“Game-changer”、“Cutting-edge”或“Revolutionary”
- 缺乏证据的模糊主张
- 未经提供上下文支持的背景或可信度声明

## 写作流程 (Writing Process)

1. 明确受众和目的。
2. 构建骨架大纲，每个章节只有一个明确目的。
3. 每个章节以证据、示例或场景开头。
4. 仅在下一句话有实际价值时才进行扩展。
5. 删除任何听起来模板化或自吹自擂的内容。

## 结构指南 (Structure Guidance)

### 技术指南
- 开篇说明读者能获得什么
- 在每个主要章节中使用代码或终端示例
- 以具体的要点 (Takeaways) 结束，而非软绵绵的总结

### 短评 / 评论文章
- 以冲突、矛盾或犀利的观察开篇
- 每个章节保持一个论点脉络
- 使用支撑观点的示例

### 时事通讯 (Newsletters)
- 确保首屏内容足够吸引人
- 将洞察与更新结合，而非流水账式的日记填充
- 使用清晰的章节标签和易于扫描的结构

## 质量把控 (Quality Gate)

交付前：
- 根据提供的来源核实事实声明
- 删除填充词和企业黑话
- 确认语调与提供的示例匹配
- 确保每个章节都增加了新信息
- 检查目标平台的格式要求
