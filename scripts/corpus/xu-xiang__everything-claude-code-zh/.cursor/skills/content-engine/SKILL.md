# 内容引擎（Content Engine）

将一个想法转化为强大的、平台原生的内容，而不是在所有地方发布同样的东西。

## 何时激活

- 编写 X 帖子或推文串（Threads）
- 起草 LinkedIn 帖子或发布更新
- 编写短视频脚本或 YouTube 讲解视频
- 将文章、播客、演示（Demos）或文档重新加工（Repurpose）为社交媒体内容
- 围绕发布、里程碑或主题构建轻量级内容计划

## 核心确认（First Questions）

明确：
- 源素材（Source Asset）：我们从什么内容进行改编
- 受众：开发者（Builders）、投资者、客户、运营者或普通受众
- 平台：X、LinkedIn、TikTok、YouTube、时事通讯（Newsletter）或多平台
- 目标：品牌感知（Awareness）、转化（Conversion）、招聘、权威性、发布支持或互动（Engagement）

## 核心规则

1. 为平台进行适配。不要在多个平台发布完全相同的文案。
2. 钩子（Hooks）比摘要更重要。
3. 每篇帖子应只承载一个清晰的想法。
4. 使用具体的细节，而不是空洞的口号。
5. 保持行动号召（Call to Action）小而清晰。

## 平台指南

### X
- 快速切入
- 每篇帖子或推文串中的每条推文只讲一个想法
- 除非必要，否则不要在正文中放链接
- 避免滥用标签（Hashtag）

### LinkedIn
- 强有力的第一行
- 短段落
- 围绕教训、结果和收获（Takeaways）进行更明确的架构

### TikTok / 短视频
- 前 3 秒必须抓住注意力（Interrupt attention）
- 围绕视觉效果编写脚本，而不仅仅是旁白
- 一个演示、一个主张、一个行动号召（CTA）

### YouTube
- 尽早展示结果
- 按章节结构化
- 每 20-30 秒刷新一次视觉画面

### 时事通讯（Newsletter）
- 提供一个清晰的视角，而不是一堆无关项目的集合
- 使各部分标题易于浏览
- 确保开篇段落具有实际价值

## 内容重加工流程（Repurposing Flow）

默认级联步骤：
1. 锚点资产（Anchor Asset）：文章、视频、演示、备忘录或发布文档
2. 提取 3-7 个原子级想法
3. 编写平台原生的变体
4. 削减输出内容之间的重复部分
5. 使行动号召（CTA）与平台意图保持一致

## 交付物

当要求提供营销活动（Campaign）时，返回：
- 核心切入点
- 针对特定平台的草稿
- 可选的发布顺序
- 可选的行动号召（CTA）变体
- 发布前所需的任何缺失输入

## 质量门禁（Quality Gate）

交付前：
- 每份草稿读起来都符合对应平台的风格
- 钩子（Hooks）强而具体
- 没有泛泛的炒作语言
- 除非要求，否则不同平台之间没有重复的文案
- 行动号召（CTA）与内容和受众匹配
