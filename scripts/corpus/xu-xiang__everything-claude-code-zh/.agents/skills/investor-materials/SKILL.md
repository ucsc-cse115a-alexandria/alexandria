# 投资者材料 (Investor Materials)

构建一致、可靠且易于辩护的面向投资者的材料。

## 何时激活 (When to Activate)

- 创建或修订路演 PPT (Pitch Deck)
- 撰写投资者备忘录 (Investor Memo) 或单页简介 (One-Pager)
- 构建财务模型 (Financial Model)、里程碑计划或资金用途表 (Use-of-funds Table)
- 回答加速器或孵化器的申请问题
- 围绕单一事实来源对齐多个融资文档

## 黄金法则 (Golden Rule)

所有投资者材料必须保持一致。

在撰写之前，建立或确认单一事实来源 (Source of Truth)：
- 牵引力指标 (Traction Metrics)
- 定价与营收假设
- 融资规模与工具 (Instrument)
- 资金用途 (Use of Funds)
- 团队简历与职位
- 里程碑与时间线 (Timelines)

如果出现数据冲突，请在起草前先予以解决。

## 核心工作流 (Core Workflow)

1. 盘点权威事实 (Canonical Facts)
2. 识别缺失的假设
3. 选择资产类型
4. 以明确的逻辑起草资产
5. 根据单一事实来源交叉检查每个数据

## 资产指南 (Asset Guidance)

### 路演 PPT (Pitch Deck)
建议流程：
1. 公司介绍 + 切入点 (Wedge)
2. 问题 (Problem)
3. 解决方案 (Solution)
4. 产品 / 演示 (Demo)
5. 市场 (Market)
6. 商业模式 (Business Model)
7. 牵引力 (Traction)
8. 团队 (Team)
9. 竞争 / 差异化 (Differentiation)
10. 诉求 (Ask)
11. 资金用途 / 里程碑
12. 附录

如果用户想要 Web 原生演示文稿，请将此技能与 `frontend-slides` 结合使用。

### 单页简介 / 备忘录 (One-Pager / Memo)
- 用一句简洁的话说明公司业务
- 展示为什么是现在 (Why Now)
- 尽早展示牵引力和证明点
- 明确表达诉求
- 确保声明易于验证

### 财务模型 (Financial Model)
包括：
- 明确的假设
- 在有用时提供悲观 (Bear) / 基础 (Base) / 乐观 (Bull) 情况
- 清晰的层级式营收逻辑
- 与里程碑挂钩的支出
- 当决策取决于假设时的敏感性分析

### 加速器申请 (Accelerator Applications)
- 针对性回答所提出的问题
- 优先考虑牵引力、洞察力和团队优势
- 避免吹嘘
- 保持内部指标与路演 PPT 及模型的一致性

## 应避免的红线 (Red Flags)

- 无法验证的声明
- 缺乏假设的模糊市场规模
- 团队角色或职位不一致
- 营收计算逻辑不严密
- 在假设脆弱的地方表现出过度的确定性

## 质量门禁 (Quality Gate)

交付前确认：
- 每个数字都符合当前的单一事实来源
- 资金用途和营收层级计算正确
- 假设清晰可见，而非被掩埋
- 叙述清晰，无浮夸语言
- 最终资产能够在合伙人会议中站得住脚
