# 文章撰写（Article Writing）

撰写听起来像真人或真实品牌的长篇内容，而非平庸的 AI 生成文本。

## 激活时机（When to Activate）

- 起草博客文章、随笔、发布公告、指南、教程或时事通讯（Newsletter）
- 将笔记、访谈实录或研究材料转化为精炼的文章
- 匹配现有创始人、运营者或示例中的品牌语调（Voice）
- 优化已撰写长篇文案的结构、节奏和论据

## 核心规则（Core Rules）

1. 以具体事物开头：示例、输出结果、轶事、数据、截图描述或代码块。
2. 先给示例，后做解释。
3. 优先使用简短、直接的句子，避免冗长堆砌。
4. 尽可能使用有来源的具体数字。
5. 严禁编造传记事实、公司指标或客户证言。

## 语调捕捉工作流（Voice Capture Workflow）

如果用户需要特定的语调，请收集以下一项或多项素材：
- 已发表的文章
- 时事通讯
- X / LinkedIn 帖子
- 文档或备忘录
- 简短的风格指南（Style Guide）

然后提取：
- 句长和节奏
- 语调是正式的、谈话式的还是犀利的
- 偏好的修辞手法，如括号、列表、片段句或提问
- 对幽默、观点和反向视角（Contrarian Framing）的接受度
- 排版习惯，如标题、项目符号、代码块和引用（Pull Quotes）

如果未提供语调参考，默认使用直接的“实干家”风格：具体、务实且不浮夸。

## 禁用模式（Banned Patterns）

删除并重写以下内容：
- 平庸的开场白，如“在当今快速演变的格局中”（In today's rapidly evolving landscape）
- 凑数过渡词，如“此外”（Moreover / Furthermore）
- 炒作词汇，如“游戏规则改变者”（game-changer）、“前沿”（cutting-edge）或“革命性”（revolutionary）
- 缺乏证据的模糊声明
- 未经提供上下文支持的传记或公信力声明

## 写作流程（Writing Process）

1. 明确受众和目的。
2. 构建骨架大纲，每个章节设定一个明确目标。
3. 每个章节都以证据、示例或场景开头。
4. 仅在下一句话有实际价值时才进行扩展。
5. 删除任何听起来像模板化或自我赞美的内容。

## 结构指南（Structure Guidance）

### 技术指南（Technical Guides）
- 开篇阐明读者将获得什么
- 在每个核心章节中使用代码或终端示例
- 以具体的干货（Takeaways）结尾，而非平淡的总结

### 随笔 / 评论文章（Essays / Opinion Pieces）
- 以冲突、矛盾或犀利的观察开篇
- 每个章节保持一个论点主线
- 使用能支撑观点的示例

### 时事通讯（Newsletters）
- 保证第一屏内容足够吸引人
- 将深刻见解与更新内容相结合，而非记流水账
- 使用清晰的章节标签和易于扫读（Skim）的结构

## 质量把关（Quality Gate）

交付前：
- 根据提供的来源核实事实陈述
- 剔除凑数词汇和企业黑话
- 确认语调与提供的示例匹配
- 确保每个章节都增加了新信息
- 检查目标平台的排版格式
