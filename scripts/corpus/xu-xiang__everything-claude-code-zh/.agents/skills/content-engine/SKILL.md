# 内容引擎 (Content Engine)

将单一想法转化为强有力的、平台原生的内容，而不是在所有地方发布同样的东西。

## 何时启用 (When to Activate)

- 编写 X (原 Twitter) 帖子或推文串 (Threads)
- 起草 LinkedIn 动态或发布更新
- 编写短视频脚本或 YouTube 讲解视频脚本
- 将文章、播客、演示 (Demos) 或文档改写为社交媒体内容
- 围绕发布、里程碑或特定主题构建轻量级内容计划

## 核心调研问题 (First Questions)

需明确：
- 源素材：我们基于什么进行改写
- 受众：开发者 (Builders)、投资者、客户、运营者或普通受众
- 平台：X、LinkedIn、TikTok、YouTube、时事通讯 (Newsletter) 或多平台
- 目标：品牌知名度、转化、招聘、权威性建立、发布支持或互动

## 核心规则 (Core Rules)

1. 适配平台特性。严禁在不同平台同步分发完全相同的文案。
2. 钩子 (Hooks) 比摘要更重要。
3. 每篇帖子应只传达一个清晰的想法。
4. 使用具体的细节，而非空洞的口号。
5. 保持号召性用语 (CTA) 小而清晰。

## 平台指南 (Platform Guidance)

### X
- 快速切入主题
- 每条帖子或推文串中的每条推文只讲一个想法
- 除非必要，否则不要将链接放在正文中
- 避免滥用话题标签 (Hashtag Spam)

### LinkedIn
- 强有力的首行
- 短段落
- 围绕经验教训、成果和要点 (Takeaways) 进行更明确的架构设计

### TikTok / 短视频
- 前 3 秒必须抓住注意力
- 围绕视觉效果编写脚本，而不仅仅是旁白
- 一个演示，一个主张，一个号召性用语 (CTA)

### YouTube
- 尽早展示结果
- 按章节结构组织
- 每 20-30 秒刷新一次视觉内容

### 时事通讯 (Newsletter)
- 提供一个清晰的视角，而不是一堆无关项目的堆砌
- 使章节标题易于快速浏览
- 确保开头段落具有实质性内容

## 内容改写流程 (Repurposing Flow)

默认级联步骤：
1. 锚点资产 (Anchor Asset)：文章、视频、演示、备忘录 (Memo) 或发布文档
2. 提取 3-7 个原子想法
3. 编写平台原生的变体内容
4. 削减不同输出之间的重复内容
5. 使号召性用语 (CTA) 与平台意图保持一致

## 交付物 (Deliverables)

当被要求策划一场活动时，返回：
- 核心切入点
- 针对特定平台的草案
- 可选的发布顺序
- 可选的号召性用语 (CTA) 变体
- 发布前所需的任何缺失输入

## 质量关卡 (Quality Gate)

交付前检查：
- 每个草案对于其所在平台都具有原生感
- 钩子 (Hooks) 强劲且具体
- 没有通用的炒作语言
- 除非有明确要求，否则不同平台之间没有重复文案
- 号召性用语 (CTA) 与内容和受众相匹配
