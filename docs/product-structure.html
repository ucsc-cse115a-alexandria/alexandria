SRP-inspired responsibility boundaries

ir owns shared data and contracts, ops owns optimization workflows, cli owns user interaction, and utils owns configuration, token helpers, and provider adapters.

DIP-inspired algorithm boundary

Core algorithms accept Embedder and merger protocols from ir. Injected adapters and test doubles can change without changing those algorithms; ops uses utils to construct defaults.

test double ↔ OpenAI adapters ↔ custom adapters
