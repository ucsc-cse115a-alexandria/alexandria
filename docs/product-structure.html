Alexandria — Product Structure

Label-free prompt optimization: four phases (represent → score → optimize → select) over one shared IR.

1. The four phases

A long prompt flows left to right through the four phases — represent → score → optimize → select — and comes out reduced. The dashed reduce loop is the one-call entry point that runs all four end to end — no setup needed, a default embedder is built on first use.

2. Architecture (layers)

The cli is a thin wrapper over ops, which composes the four pluggable features on top of the utils shell. utils (imperative shell) and ir (pure shared contract) form the foundation pair at the bottom — and ir may be imported from every layer while depending on nothing else in alexandria. Layering is enforced by five import-linter contracts.
