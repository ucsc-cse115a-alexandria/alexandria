Alexandria — Product Structure

Label-free prompt optimization: four phases (represent → score → optimize → select) over one shared IR.

1. The four phases (represent → score → optimize → select)

Read it as a matrix: the columns are the four phases, each row is one sentence (s1 s2 s3) tracked all the way through. The redundant s3 is scored high, proposed for deletion by Optimize, then actually removed by Select once its drift gate confirms the shorter prompt barely changes meaning.

s1 & s3 are near-duplicates (amber, cosine 0.95 ≥ threshold 0.85); s2 is unique (slate, 0.40). Optimize only proposes deleting the more-redundant sentence of a pair (ranking, not deciding). Select applies proposals highest-confidence first and keeps each only while the reduced prompt stays within drift_budget (1%) cosine distance of the original Document embedding. apply is the core primitive Select uses to rewrite the tree.

2. Module layout & dependency rule

core is the contract and depends on nothing; phases import only core; runtime is the imperative shell that builds the model and drives the phases; cli is a leaf. Layering enforced by import-linter.
